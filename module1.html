Web Programming Basics

Understanding the Internet, the World Wide Web, and how web applications are built

Unit 1

Definitions of Web Programming, the Internet and WWW

Web Programming refers to the process of writing, testing, and maintaining the code that makes websites and web applications function. It encompasses both the visual front-end that users interact with and the back-end logic that runs on servers.

The Internet is a global system of interconnected computer networks that communicate using the Internet Protocol (IP) suite. It is the physical and logical infrastructure — the roads — that data travels on.

The World Wide Web (WWW) is a system of interlinked hypertext documents and resources accessed via the Internet through a web browser. The Web is a service that runs on top of the Internet — one of many, alongside email, FTP, and instant messaging.

Unit 2

Internet Services and Communication Protocols

The Internet provides a wide range of services beyond the Web. Each service relies on specific communication protocols — standardised rules that define how data is sent and received between systems.

Common Internet Services

  • World Wide Web (HTTP/HTTPS) — browsing hypertext documents and web apps
  • Email (SMTP, POP3, IMAP) — sending and receiving electronic messages
  • File Transfer (FTP/SFTP) — uploading and downloading files between systems
  • Domain Name System (DNS) — translating domain names to IP addresses
  • Instant Messaging & VoIP — real-time text, voice, and video communication
  • Streaming (RTSP/WebRTC) — audio and video delivery in real time

Unit 3

Web Application Architecture and the Network Model

A web application is structured in layers. The most widely used model is the Three-Tier Architecture:

  1. Presentation Layer (Client/Front-end) — what the user sees in their browser. Built with HTML, CSS, and JavaScript.
  2. Application Layer (Business Logic/Back-end) — processes user requests and communicates with the database. Built with PHP, Node.js, Python, etc.
  3. Data Layer (Database) — stores and retrieves data. Examples: MySQL, MongoDB, PostgreSQL.
How a request works: A user types a URL → browser sends an HTTP request → server processes it → retrieves data from the database → sends an HTTP response back → browser renders the page.

Unit 4

Web Development Life Cycle

The Web Development Life Cycle (WDLC) is a structured process for planning, building, and maintaining a web application.

  1. Requirement Gathering & Planning — Define project goals, target audience, features, timeline, and budget.
  2. System Design — Create wireframes, UI mockups, database schemas, and architecture diagrams.
  3. Development — Write the actual code — front-end (HTML/CSS/JS) and back-end (server & database logic).
  4. Testing & Quality Assurance — Test for bugs, cross-browser compatibility, responsiveness, and security.
  5. Deployment — Upload the web application to a live server and make it accessible to users.
  6. Maintenance & Updates — Monitor performance, fix bugs, add new features, and keep the application secure.
